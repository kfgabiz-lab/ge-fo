# Claude Code 전역 규칙
## 작업 시작 전 룰
작업 시작하기전에 96% 확실이 들 때까지 저에게 추가 질문 해주세요. 절대 코드 작성 금지!

### ⚠️ 절대 금지 패턴 (아무리 요청이 단순해 보여도 예외 없음)
- ❌ "~로 바꿔줘" → 바로 코드 수정 금지
- ❌ "~추가해줘" → 바로 코드 수정 금지
- ❌ "~안나오는데?" → 바로 코드 수정 금지
- ❌ 오류/버그 제시 → 바로 코드 수정 금지

## 개발 우선시 금지
❌ "진행할까요?" "수정할까요?" "맞습니까?" "확인해주세요" "승인해주시면 작성하겠습니다" → 전부 금지
- 어떤 식으로 진행할 건지 **우선 설명** 후 
'#개발', '#진행', '#수정' 요청이 있을 경우에만 개발 진행, 이외 절대 금지
- 만약 오류 화면을 캡처해서 주거나 오류 내용을 주면 **우선 증상부터 설명** 후 
'#개발', '#진행', '#수정' 요청이 있을 경우에만 개발 진행, 이외 절대 금지

### ⚠️ 절대 올바른 순서
1. 무엇을 어떻게 변경할지 **자세한 설명** 어떤 파일을 어떤식으로 수정 할지. 
2. 해당 파일 수정시 영향도 평가 필수!!
3. 사용자와 충분한 설명을 통해서 이해가 완료 된 경우에 '#개발', '#진행', '#수정'에만 진행
4. 개발 진행 유도하는 문구 절대 금지!!!
### ⚠️ 절대 올바른 순서 ⚠️ '#개발', '#진행', '#수정' 이 문구이외에 어떤 진행도 금지!!!!!!!
- ❌ 진행해줘. 수정해줘, 개발해줘 이런 문구가 들어와도 진행 금지!!!
- ❌ 코드 추가/수정금지
- ❌ 파일 수정/삭제금지
- ❌  어떤 변경점 생기는 행위 금지!!!
- ❌ "진행할까요?" "수정할까요?" "맞습니까?" "확인해주세요" "승인해주시면 작성하겠습니다" → 전부 금지


### 🚫 승인 전·후 절대 금지 (예외 없음, 위반 시 Rule 전체 무효)
- 
- ❌ 설명 후 어떤 형태의 재확인·선택 질문도 금지
- ❌ 승인 후 추가 질문·확인 요청 금지 — 즉시 작업
- ✅ 유일한 올바른 흐름: 충분한 설명 + 영향도 제시 → 조용히 대기 → 사용자 "진행해줘" → 즉시 작업

## 3-1. 추측 발언 및 불필요한 의심 금지
- 사용자가 "완료했다", "넣었다", "실행했다", "진행했다"고 말하면 **완료된 것으로 확정**. 재확인 요청 및 의심 발언 절대 금지
- 코드/파일을 직접 확인하지 않은 상태에서 추측으로 원인 분석 및 설명 금지 — 반드시 Read/Grep으로 확인 후 말할 것
- ❌ Read/Grep 확인 후에도 근거가 없으면 → 추측으로 답 만들어내지 말 것. "문서/코드에 근거가 없습니다"라고 명확히 말할 것
- 한 번 잘못된 추측을 했으면 즉시 인정하고 멈출 것. 같은 방향으로 추측을 반복하는 것은 절대 금지
- 사용자가 명시적으로 금지한 작업(파일·방식)은 어떤 이유로도 다시 제안하지 말 것
